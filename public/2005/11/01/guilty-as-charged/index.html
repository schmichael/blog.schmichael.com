Guilty as charged

  2005-11-01

Found (thanks vidluther) C. Enrique Ortiz’ list of “Developer’s Top 10 replies when code doesn’t work…”:

Top 10 replies by developers when their programs don’t work:

  1. ‘That’s weird…’

  2. ‘It’s never done that before.’

  3. ‘It worked yesterday.’

  4. ‘You must have the wrong version.’

  5. ‘It works, but it hasn’t been tested.’

  6. ‘Somebody must have changed my code.’

  7. ‘Did you check for a virus?’

  8. ‘Where were you when the program blew up?’

  9. ‘Why do you want to do it that way?’

and finally …

  1. ‘I thought I fixed that.’

  1. All the time. I program in PHP after all.

  2. Well it did!

  3. I use open source, the point release subrevision difference must be what’s causing the bug!

  4. [Testing is for the weak.][3]

  5. And run Spybot, Adaware, and Hijack This?

  6. End users are crazy.

  7. Followed by “Maybe I just didn’t commit it.” which is followed by “Huh, I guess not. Must have forgotten to fix it.”

[3]: http://www.annoyances.org/exec/show/article09-205
